About Me

Hi, I'm Daniel McCluskey, a Graduate Computer Games Programmer, working as a Research Assistant for the Creative Computing Courses at the University of Gloucestershire, currently focusing on writing course material for Artificial Intelligence, Data Science and Games Programming related courses.

Skilled with the Unity engine for mobile and PC development using C#, Unreal Engine development for games using C++, and experience with the programming API and Engine integrations of the FMOD audio framework. I also have a keen interest in Artificial Intelligence and Machine Learning for games and have gained experience in AI techniques including behaviour trees, state machines, goal-oriented action planners, pathfinding algorithms and basic Neural Network implementations for games during my time at University as both a student and a Research Assistant.

Self-taught knowledge of website development using HTML and CSS, using frameworks such as Bootstrap and Skeleton. Experience with using Python for data science in tandem with libraries such as FastAI, Pandas, Matplotlib and Numpy. Knowledge of ECS through practical implementation in C++ and Amateur experience with OpenGL and GLSL shader programming in C++.

Unity - AI Spy vs Guard

This Unity project provides a scenario including reactive ‘Guard’ AI agents and a planning based ‘Spy’ AI agent. The role of the ‘Guard’ agents is to patrol a given area and react to the ‘Spy’ in an attempt to catch them. The role of the ‘Spy agent is to avoid the ‘Guard’ agents to successfully achieve its goals (Find a key and get to the exit).

This project uses a Goal Oriented Action Planning method to drive the guards, this allows them to make their own decisions by pathfinding through their available actions to reach a goal.

A document describing the features and actions implemented for this can be found here, or the video below shows the Spy AI Agent making a successful runthrough of the level.

Under the Bus (WIP)

Under the Bus is a unique party game, created in Unity, that connects several android devices to a central PC, allowing users to play mini-games that utilise the gyroscope, camera and touch-screen. Up to six devices (players) can connect to the server PC. The PC will show the main game, while the android devices act as controllers. There is a central hub in which players will be able to choose the next mini-game to play (faced with 6 choices randomly chosen from the mini-game list), which can include 1vWorld, 3v3 and PvP. Players can customise their characters here, based on attire won from playing the game.

My role within the team that made this game was a gameplay programmer, working on the minigames and general back end systems.
